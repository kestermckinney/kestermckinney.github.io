Overview

What it solves

ProjectNotes is built around the actual rhythm of project management work. Instead of scattering information across email, spreadsheets, folders, and disconnected notes, it brings together meeting notes, action items, risks, issues, change requests, work items, and project metrics in one application.

The product is designed for project managers working across multiple teams who need a reliable operating hub for project communication, tracking, and reporting.

The application also emphasizes automation. Python-based extensions let the platform integrate with external systems and automate repeatable project tasks such as exporting notes, generating reports, creating meeting artifacts, and connecting with enterprise tools.

A shared core library and mobile work indicate the product is evolving beyond a single desktop workflow into a broader project operations platform.

Technical Story

How it is built

Desktop foundation

The desktop application is built in Qt and C++17 with a substantial UI and data model surface area for managing projects, people, clients, notes, trackers, and reporting workflows.

Shared core architecture

`ProjectNotesCore` separates the shared data layer from the desktop shell, allowing the same business model to support mobile work and future expansion.

Extensibility

Python plugins and background worker scripts make it possible to adapt the system to organization-specific workflows and integrations without reshaping the entire product.

Why it matters

A practical product for project leaders

ProjectNotes reflects a point of view that project management software should support the real work of coordination, accountability, and communication. It is not just a place to store notes. It is a working environment built around the reality that projects succeed when information is easy to capture, easy to revisit, and easy to turn into action.
